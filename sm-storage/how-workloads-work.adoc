= SANtricity 소프트웨어에서 워크로드가 작동하는 방식
:allow-uri-read: 
:icons: font
:imagesdir: ../media/


[role="lead"]
볼륨을 생성할 때 특정 애플리케이션에 대한 스토리지 어레이 구성을 사용자 지정할 워크로드를 선택합니다.

워크로드는 애플리케이션을 지원하는 스토리지 객체입니다. 애플리케이션별로 하나 이상의 워크로드 또는 인스턴스를 정의할 수 있습니다.



== 워크로드 생성

특정 애플리케이션과 연결되지 않은 워크로드를 생성하려면 수동으로 지정해야 합니다. 또한 볼륨 추가 페이지를 사용하여 볼륨 구성을 수동으로 지정해야 합니다.



== 애플리케이션 및 워크로드 뷰

애플리케이션 및 워크로드를 보려면 SANtricity 시스템 관리자를 시작하십시오. 이 인터페이스를 통해 다음과 같은 몇 가지 방법으로 애플리케이션별 워크로드와 관련된 정보를 볼 수 있습니다.

* 볼륨 페이지에서 *애플리케이션 및 워크로드* 탭을 선택하면 스토리지 시스템의 볼륨이 워크로드 및 해당 워크로드와 연결된 애플리케이션 유형별로 그룹화되어 표시됩니다.
* Performance 페이지에서 *Applications and workloads* 탭을 선택하면 논리 객체의 성능 지표(지연 시간, IOPS, MB)를 볼 수 있습니다. 객체는 애플리케이션 및 관련 워크로드별로 그룹화됩니다. 이러한 성능 데이터를 정기적으로 수집하면 기준 측정값을 설정하고 추세를 분석할 수 있으며, 이는 I/O 성능 관련 문제를 조사하는 데 도움이 될 수 있습니다.
